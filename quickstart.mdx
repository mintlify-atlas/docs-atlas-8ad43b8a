---
title: "Get started with BACLAB: Run your first WGS analysis"
description: "Learn how to open the BACLAB Colab notebook, install dependencies, configure your inputs, and run your first bacterial WGS analysis end to end."
---

This guide walks you through opening the BACLAB notebook in Google Colab, installing all required tools, configuring your input data, and running the full trimming → QC → assembly pipeline. By the end you will have a ZIP archive of assembled genomes and QC reports ready to download.

<Steps>
  <Step title="Open the notebook">
    Open the BACLAB notebook in Google Colab:

    [https://colab.research.google.com/drive/1MC1bg80356yF6ivsmNk4wxjYUu7elNRh?usp=sharing](https://colab.research.google.com/drive/1MC1bg80356yF6ivsmNk4wxjYUu7elNRh?usp=sharing)

    Sign in with your Google account if prompted, then click **Copy to Drive** to save your own editable copy before making any changes.
  </Step>

  <Step title="Run the installation cells">
    The first cells in the notebook install Conda (via Miniforge) and create the `baclab_wgs` environment with all required tools:

    ```text
    trimmomatic  fastqc  multiqc  unicycler  spades  openjdk  pandas
    ```

    Run these cells in order and wait for them to complete. On a fresh Colab session this typically takes **3–5 minutes**. You only need to repeat this step if your runtime is recycled.

    <Tip>
      If you see a prompt asking to restart the runtime after installation, click **Restart** and then continue running the cells below the installation section.
    </Tip>
  </Step>

  <Step title="(Optional) Mount Google Drive">
    Colab's `/content` filesystem is ephemeral — it is wiped when the session ends. Mounting Google Drive gives your databases and output files persistence across sessions.

    Run the Drive mount cell and, when prompted, authorise access. Then set `DB_ROOT` to a path inside your Drive:

    ```python
    DB_ROOT = "/content/drive/MyDrive/baclab_databases"
    ```

    This ensures downloaded databases (ResFinder, MLST, etc.) are not lost between sessions.
  </Step>

  <Step title="Configure your input mode">
    BACLAB supports two input modes. Set the `MODE` variable to one of the following:

    **`SINGLE_SAMPLE`** — run one sample by providing explicit FASTQ paths:

    ```python
    MODE = "SINGLE_SAMPLE"

    # Paired-end
    R1_PATH = "/content/sample_R1.fastq.gz"
    R2_PATH = "/content/sample_R2.fastq.gz"

    # Single-end (use instead of R1/R2)
    # SE_PATH = "/content/sample.fastq.gz"
    ```

    **`SAMPLESHEET`** — run multiple samples in batch by pointing to a folder of FASTQs and a CSV/TSV samplesheet:

    ```python
    MODE = "SAMPLESHEET"
    DATA_DIR = "/content/drive/MyDrive/my_fastqs"
    SAMPLESHEET_PATH = "/content/drive/MyDrive/samplesheet.csv"
    ```

    See [Input modes](/pipeline/input-modes) for samplesheet format details and examples.
  </Step>

  <Step title="Run the pipeline">
    With your inputs configured, run the three pipeline cells in order:

    1. **Trimming** — Trimmomatic clips adapters and removes low-quality bases from all samples.
    2. **FastQC / MultiQC** — FastQC generates per-sample reports; MultiQC summarises them into `multiqc_report.html`.
    3. **Assembly** — Unicycler assembles PE samples; SPAdes (`--careful`) assembles SE samples.

    Each cell prints progress logs as it runs. Assembly is the most resource-intensive step — runtime depends on genome size and the number of samples.
  </Step>

  <Step title="Download your results">
    After assembly completes, the final cell packages everything into a ZIP and triggers an automatic browser download:

    ```text
    baclab_bacterial_pipeline_results_<timestamp>.zip
    ```

    The archive contains trimmed FASTQs, QC reports, assembled FASTA files, and per-sample logs. See [Output structure](/output/structure) for a full breakdown of the directory layout inside the ZIP.
  </Step>
</Steps>

<Note>
  Colab's free tier has memory and storage limits that can cause sessions to crash during large assemblies. If your session crashes mid-run, reduce `THREADS_ASSEMBLY`, run fewer samples at a time, or upgrade to Colab Pro for more resources. See [Tips and notes](/tips-and-notes) for more guidance.
</Note>
